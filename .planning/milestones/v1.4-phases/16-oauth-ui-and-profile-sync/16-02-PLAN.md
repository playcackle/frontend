With a conditional that renders `<Image>` when `avatar_url` exists, falling back to initials:
```tsx
{/* AFTER */}
{profile.avatar_url ? (
  <Image
    src={profile.avatar_url}
    alt={`${profile.name}'s avatar`}
    width={72}
    height={72}
    className={styles.avatar}
  />
) : (
  <div className={styles.avatar}>
    {(profile.name || "?")[0].toUpperCase()}
  </div>
)}
```

Key details:
- `width={72}` and `height={72}` match the `.avatar` CSS class dimensions (72px x 72px)
- The `className={styles.avatar}` applies the same border-radius: 50%, border, and box-shadow from page.module.css — the Image will render as a circle with the neon border
- The `.avatar` CSS has `border-radius: 50%` which clips the image to a circle. The Image component's intrinsic size (72x72) matches the CSS width/height so there's no layout shift
- If the backend does not return `avatar_url` (field is undefined at runtime), JavaScript's optional chaining in the profile type means it evaluates as falsy and renders the initials fallback. No runtime error.
- Do NOT add `unoptimized` prop — let Next.js Image optimizer handle the CDN images (this is why SETUP-05 remotePatterns exist)
